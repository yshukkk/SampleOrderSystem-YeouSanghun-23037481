# SampleOrderSystem

반도체 시료(Sample) 생산·주문을 관리하는 콘솔 애플리케이션입니다. 시료 등록부터 주문 접수, 승인/거절, 생산, 출고, 모니터링까지 전체 흐름을 하나의 메뉴 기반 프로그램에서 처리합니다. 자세한 기능 명세는 [`PRD.md`](PRD.md), 구현 계획은 [`PLAN.md`](PLAN.md)를 참고하세요.

## 이 프로그램이 하는 일

- **시료 관리**: 시료(반도체 샘플)를 ID와 함께 등록하고, 목록/검색으로 조회
- **주문 접수 및 승인/거절**: 고객 주문을 접수하고, 재고 상황에 따라 즉시 확정하거나 생산 큐에 등록
- **생산 라인**: 재고가 부족해 생산이 필요한 주문을 FIFO로 처리하며, 실제 생산에 걸리는 시간이 지나면 자동으로 완료 처리(재고 반영)
- **출고 처리**: 확정된(CONFIRMED) 주문을 출고 처리
- **모니터링**: 상태별 주문 수와 시료별 재고 상태(여유/부족/고갈)를 조회

데이터는 `data/samples.json`, `data/orders.json`에 JSON으로 저장되어 프로그램을 껐다 켜도 유지됩니다.

## 요구 사항

- Python 3.10 이상
- 외부 런타임 의존성 없음 (표준 라이브러리만 사용)
- 테스트 실행용: `pytest` (`requirements-dev.txt`에 명시)

## 메인 메뉴 구조

```
----- SampleOrderSystem 메인 메뉴 -----
등록 시료 수: N | 전체 주문 수: N | 생산라인 대기: N
1. 시료 관리
2. 주문 (접수/승인/거절)
3. 생산 라인
4. 출고 처리
5. 모니터링
6. 종료
```

메인 메뉴의 요약 줄은 매번 최신 데이터를 다시 읽어 표시합니다. "전체 주문 수"는 이미 끝난 주문(RELEASED/REJECTED)을 제외한, 현재 진행 중인 주문만 셉니다.

각 하위 메뉴는 항상 마지막 항목이 **"뒤로가기"**(메인 메뉴로 복귀)이며, 프로그램을 완전히 종료하는 **"종료"**는 메인 메뉴에만 있습니다.

### 1. 시료 관리

```
----- 시료 관리 -----
1. 시료 등록
2. 시료 목록 조회
3. 시료 검색
4. 뒤로가기
```

- **시료 등록**: `ID → 이름 → 평균 생산시간 → 수율` 순서로 입력. 시료 ID는 자동 부여가 아니라 직접 지정하며, 이미 사용 중인 ID는 거부됩니다. 재고는 항상 0으로 시작합니다.
- **시료 검색**: 한 줄로 `ID: <숫자>` 또는 `이름: <검색어>` 형식으로 입력합니다.

### 2. 주문 (접수/승인/거절)

```
----- 주문 (접수/승인/거절) -----
1. 주문 접수
2. 접수된 주문 목록
3. 주문 승인
4. 주문 거절
5. 뒤로가기
```

- **주문 접수**: `시료 ID → 고객명 → 주문 수량` 순서로 입력. 등록되지 않은 시료 ID는 거부됩니다.
- **접수된 주문 목록**: 아직 승인/거절되지 않은(RESERVED) 주문만 표시됩니다.
- **주문 승인**: 재고가 충분하면 즉시 CONFIRMED(출고 대기), 부족하면 생산 큐에 등록되고 PRODUCING 상태가 됩니다.

### 3. 생산 라인

```
----- 생산 라인 -----
1. 생산 라인 현황
2. 대기 주문 확인
3. 뒤로가기
```

- 생산은 하나의 라인에서 한 번에 하나씩 진행됩니다(FIFO).
- 목표 생산 시간(부족분·수율로 계산된 총생산시간)이 실제로 지나면 **자동으로** 완료되어 해당 주문이 CONFIRMED로 바뀌고 재고에 반영됩니다 — 별도로 "완료 처리"를 선택할 필요가 없습니다.

### 4. 출고 처리

```
----- 출고 처리 -----
(CONFIRMED 상태 주문 목록이 자동으로 표시됨)
출고할 주문 번호를 입력하세요 (뒤로가기: 0):
```

메뉴 선택 없이, 들어가자마자 출고 가능한(CONFIRMED) 주문 목록이 바로 보이고 주문 번호만 입력하면 됩니다. `0`을 입력하면 아무것도 출고하지 않고 메인 메뉴로 돌아갑니다.

### 5. 모니터링

```
----- 모니터링 -----
1. 주문량 확인
2. 재고량 확인
3. 뒤로가기
```

- **주문량 확인**: RESERVED/CONFIRMED/PRODUCING/RELEASED 상태별 주문 수. REJECTED는 유효한 주문이 아니므로 집계에서 제외됩니다.
- **재고량 확인**: 시료별 재고와 "주문 대비" 상태 — **여유**(재고가 진행 중인 주문 수요보다 충분) / **부족**(재고보다 수요가 많음) / **고갈**(재고가 정확히 0).

## 사용 예시

아래는 시료 등록부터 출고까지 한 번의 전체 흐름을 콘솔 입출력으로 보여주는 예시입니다.

### 예시 1 — 재고가 충분해서 바로 출고까지 가는 경우

```
메인 메뉴 → 1 (시료 관리) → 1 (시료 등록)
> (시료 등록 입력 순서: ID -> 이름 -> 평균 생산시간 -> 수율)
ID 입력: 1
이름 입력: 웨이퍼-A
평균 생산시간 입력: 2.0
수율 입력: 0.9
> 시료가 등록되었습니다: ID=1

메인 메뉴 → 1 (시료 관리) → 4 (뒤로가기) → 메인 메뉴 → 2 (주문) → 1 (주문 접수)
> (주문 접수 입력 순서: 시료 ID -> 고객명 -> 주문 수량)
시료 ID: 1
고객명: 홍길동
주문 수량: 5
> 주문이 접수되었습니다: ID=1 상태=RESERVED
```

이 시점에서 시료 재고는 0이라 그대로 승인하면 생산이 필요합니다. 재고를 미리 채워둔 경우(예: 이전에 다른 주문으로 생산이 끝나 재고가 쌓인 상태)라면:

```
메인 메뉴 → 2 (주문) → 3 (주문 승인)
> (승인할 주문 번호를 입력하세요)
주문 번호: 1
> 주문이 승인되었습니다: ID=1 상태=CONFIRMED   ← 재고 충분 → 즉시 확정

메인 메뉴 → 4 (출고 처리)
----- 출고 처리 -----
ID | 시료ID | 고객명 | 수량 | 상태
-------------------------------------
1 | 1 | 홍길동 | 5 | CONFIRMED
출고할 주문 번호를 입력하세요 (뒤로가기: 0): 1
> 출고 처리되었습니다: ID=1 상태=RELEASED
```

### 예시 2 — 재고 부족 → 생산 → 자동 완료 → 출고

```
메인 메뉴 → 2 (주문) → 3 (주문 승인)
> (승인할 주문 번호를 입력하세요)
주문 번호: 2
> 생산 큐에 등록되었습니다: 부족분=5 실생산량=6 총생산시간=12.0
> 주문이 승인되었습니다: ID=2 상태=PRODUCING

메인 메뉴 → 3 (생산 라인) → 1 (생산 라인 현황)
> 생산 라인 대기 중인 항목 수: 1
> 생산 중: 주문 ID=2 시료ID=1 목표생산량=6 현재까지 생산량=2
```

(총생산시간만큼 실제 시간이 지난 뒤 다시 메뉴에 들어가거나 메인 메뉴로 돌아가면, 별도 조작 없이 자동으로 완료됩니다.)

```
메인 메뉴로 돌아가면 자동 완료 메시지가 출력됩니다:
> 생산이 완료되었습니다: 주문 ID=2 상태=CONFIRMED 재고 증가=6 (시료ID=1)

메인 메뉴 → 4 (출고 처리) → 해당 주문 번호 입력 → 출고 완료
```

### 예시 3 — 시료 검색

```
메인 메뉴 → 1 (시료 관리) → 3 (시료 검색)
> (시료 검색 입력 형식: 'ID: <숫자>' 또는 '이름: <검색어>')
검색어: ID: 1
ID | 이름 | 평균생산시간 | 수율 | 재고
-------------------------------------
1 | 웨이퍼-A | 2.0 | 0.9 | 6
```

```
검색어: 이름: 웨이퍼
ID | 이름 | 평균생산시간 | 수율 | 재고
-------------------------------------
1 | 웨이퍼-A | 2.0 | 0.9 | 6
```

## 아키텍처 개요

- **Model / View / Controller** 계층 분리 (`src/sampleordersystem/{model,view,controller}`), 상태 전이 규칙은 콘솔 I/O 없이 단위 테스트 가능한 순수 함수로 `model/order.py`에 위치
- **영속성**: `persistence/json_repository.py`가 JSON 파일 기반 CRUD 제공, 시료/주문 각각 별도 파일
- **생산 큐**: FIFO, 벽시계(wall-clock) 기준 실시간 게이팅 — 프로세스를 재시작해도 실제 경과 시간이 보존됨
